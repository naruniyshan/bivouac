1. 設計內頁時，要先判斷共用部份（首頁）

2. 設計多頁網頁，建立範本較好管理
    2-1 Dreamweaver
    2-2 Javascript
    2-3 Vue
    2-4 PHP（資料庫語言）

3. 共用框架，需設定不同的 class 名稱，樣式表內容才不會有衝突
    3-1 因為 css 是由上往下讀取
    3-2 不同 html 檔案，讀取同一個 css 檔時，若沒有不同名稱管理，會基於第 1 點的特性，造成後面樣式直接覆蓋前面樣式

表格  <table></table>
列 <tr></tr>
欄 <td></td>
合併儲存格
垂直合併（同一欄位，不同一列） rowspan
水平合併（同一列，不同欄位）   colspan

scss 繼承（集中管理相同的樣式）
1. 可以不用命名 class or id
缺點
1. 無法跨檔案應用
2. 名稱不能相同
